### GAP #1 — API-key-auth POST inserts block without junction

`app/api/api-management/campus-living/blocks/route.ts` (lines 44–59):

```ts
export const POST = withAuth(async (request, auth) => {
  const institutionId = auth.institutionId;
  if (!institutionId) return errorResponse('API key must be associated with an organization', 400);
  const body = await request.json();
  const { data, error } = await (auth.supabase as any)
    .from('hostel_blocks')
    .insert({ ...sanitizeBody(body), institution_id: institutionId })
    .select()
    .single();
  if (error) throw error;
  return createdResponse(data);
}, { allowApiKey: true, requiredPermission: 'write' });
```

Same bug class as PR #746 (UI side). API consumers creating blocks via API key get RLS-invisible rows on the junction-aware reader path.

## 4. Residual service-layer concerns

### Concern A — `hostel-block-service.ts::updateBlock` doesn't sync junction on `institution_id` change

If a block's `institution_id` is updated, the M2M `is_primary=true` row's `institution_id` is NOT updated → drift. Probably rare in practice (UI doesn't seem to expose institution-change), but worth a guard. Logged here, not fixed in this work item.

### Concern B — Deleted blocks leave junction rows? No — `hostel_block_institutions.block_id REFERENCES hostel_blocks(id) ON DELETE CASCADE`. Clean.

### 5.1 Backfill missing junction rows for any blocks created via the API-key POST (or any other path that bypassed the junction):

```sql
-- DRY-RUN: count orphans first
SELECT COUNT(*)
FROM public.hostel_blocks hb
WHERE hb.institution_id IS NOT NULL
  AND NOT EXISTS (
    SELECT 1 FROM public.hostel_block_institutions hbi
    WHERE hbi.block_id = hb.id
      AND hbi.is_primary = true
  );

-- Backfill
INSERT INTO public.hostel_block_institutions (block_id, institution_id, is_primary)
SELECT hb.id, hb.institution_id, true
FROM public.hostel_blocks hb
WHERE hb.institution_id IS NOT NULL
  AND NOT EXISTS (
    SELECT 1 FROM public.hostel_block_institutions hbi
    WHERE hbi.block_id = hb.id
  )
ON CONFLICT (block_id, institution_id) DO NOTHING;
```
